### 2. 锁文件数据结构

#### 项目级锁文件结构 (LocalSkillLockFile)

```typescript
export interface LocalSkillLockFile {
  version: number;
  skills: Record<string, LocalSkillLockEntry>;
}

export interface LocalSkillLockEntry {
  source: string;           // 来源
  sourceType: string;       // 提供者类型 (local, local-zip, github, pingancoder-api)
  computedHash: string;     // 技能文件夹 SHA256 哈希
  version?: string;         // 版本（如果可用）
}
```

**示例：**
```json
{
  "version": 1,
  "skills": {
    "test-skill": {
      "source": "/tmp/test-skill",
      "sourceType": "local",
      "computedHash": "f99aaee997b27b314fdf30bd1fec486b2bbc28facd014ba4cf96c1e99961b030"
    },
    "zip-test": {
      "source": "/tmp/zip-test.zip",
      "sourceType": "local-zip",
      "computedHash": "669ee540b51189068cdee056848875ea83e6c453bb6a46b86ce4d68a22ba44dd"
    }
  }
}
```

#### 全局锁文件结构 (SkillLockFile)

```typescript
export interface SkillLockFile {
  version: number;
  skills: Record<string, SkillLockEntry>;
  dismissed?: DismissedPrompts;        // 忽略的提示
  lastSelectedAgents?: string[];       // 最后选择的代理
}

export interface SkillLockEntry {
  source: string;                      // 规范化的来源标识符
  sourceType: string;                  // 提供者类型
  sourceUrl: string;                   // 原始 URL
  skillPath?: string;                  // 子路径（如果适用）
  skillFolderHash: string;             // 技能文件夹哈希
  installedAt: string;                 // 安装时间 (ISO 8601)
  updatedAt: string;                   // 更新时间 (ISO 8601)
  version?: string;                    // 版本（如果提供者支持）
  pluginName?: string;                 // 插件名称
}
```

**示例：**
```json
{
  "version": 3,
  "skills": {
    "find-skills": {
      "source": "vercel-labs/skills",
      "sourceType": "github",
      "sourceUrl": "https://github.com/vercel-labs/skills.git",
      "skillPath": "skills/find-skills/SKILL.md",
      "skillFolderHash": "c2f31172b6f256272305a5e6e7228b258446899f",
      "installedAt": "2026-01-28T03:35:58.486Z",
      "updatedAt": "2026-01-28T03:35:58.486Z"
    }
  },
  "dismissed": {
    "findSkillsPrompt": true
  },
  "lastSelectedAgents": ["gemini", "opencode", "pingancoder"]
}
```

### 3. 规范位置路径

| 安装类型 | 规范位置 | 常量定义 |
|---------|---------|---------|
| **项目级** | `{cwd}/.agents/skills/` | `AGENTS_DIR = '.agents'` + `SKILLS_SUBDIR = 'skills'` |
| **全局级** | `~/.agents/skills/` | `AGENTS_DIR = '.agents'` + `SKILLS_SUBDIR = 'skills'` |

**代码实现：**
```typescript
// installer.ts
export function getCanonicalSkillsDir(global: boolean, cwd?: string): string {
  const baseDir = global ? homedir() : cwd || process.cwd();
  return join(baseDir, AGENTS_DIR, SKILLS_SUBDIR);
}

// 实际路径：
// - 项目级: {cwd}/.agents/skills
// - 全局级: ~/.agents/skills
```

## 关键设计决策

### 1. 为什么项目级锁文件在根目录？

```
项目根目录/
├── skills-lock.json      ← 项目级锁文件（直接在根目录）
├── .agents/
│   └── skills/           ← 规范位置
└── package.json
```

**原因：**
- 便于版本控制（可以提交到 Git）
- 便于用户查看和修改
- 与项目配置文件保持一致的位置

### 2. 为什么全局锁文件在 `~/.agents/` 目录？

```
~/.agents/
├── .skill-lock.json      ← 全局锁文件
└── skills/               ← 全局技能目录
    └── skill-name/
```

**原因：**
- 与全局规范位置 `~/.agents/skills/` 保持一致
- 集中管理全局技能相关文件
- 避免污染用户主目录

### 3. sourceType 字段的值

根据代码实现和实际锁文件，`sourceType` 支持以下值：
- `local` - 本地路径
- `local-zip` - 本地 Zip 文件
- `github` - GitHub 仓库
- `pingancoder-api` - 内网 API
